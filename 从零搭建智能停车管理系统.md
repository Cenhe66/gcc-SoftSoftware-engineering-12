## 一、课程概览与项目架构

### 1.1 我们先来看看要盖一栋什么样的"大楼"

这个智能停车管理系统，本质上是一个 **前后端分离 + 多端覆盖** 的完整项目。我们可以把它想象成一栋三层小楼：

| 楼层 | 模块 | 技术栈 | 作用 |
|------|------|--------|------|
| 第一层（地基） | `parking-server` | Java + Spring Boot + MySQL + Redis | 提供所有数据接口和业务逻辑 |
| 第二层（门面） | `parking-admin` | Vue 3 + Vite + Element Plus | 管理员使用的数据大屏和后台管理系统 |
| 第三层（触手） | `parking-miniapp` | 微信小程序原生框架 | 普通用户手机端使用的停车小程序 |

**核心业务流程：**
- 用户打开微信小程序 → 查看附近停车场 → 预约/导航/支付/共享车位
- 管理员打开Web后台 → 查看数据大屏 → 管理停车场/用户/记录/系统配置
- 后端服务统一处理所有数据，并通过 WebSocket 提供实时车位状态推送

### 4.1 为什么选择 Spring Boot？

同学们，Spring Boot 是 Java 企业级开发的事实标准。它帮我们自动配置好了绝大多数东西，让我们可以专注于写业务代码，而不是折腾 XML 配置。

mkdir -p src/test/java
```

### 4.4 创建启动类

### 4.7 数据库表结构设计

2. `parking_lot` - 停车场表
3. `parking_space` - 车位表
4. `parking_record` - 停车记录表
5. `reservation` - 预约订单表

**老师提醒：** 建表时注意三个细节：

- 使用 `deleted` 字段做逻辑删除，永远不要物理删除数据！

    ]
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

// 路由守卫 - 登录校验
router.beforeEach((to, from, next) => {
  if (to.meta.title) {
    document.title = to.meta.title + ' - 智能停车管理系统'
  }
  if (!to.meta.public) {
    const token = localStorage.getItem('token')
    if (!token) {
      next('/login')
      return
    }
  }
  next()
})

export default router
```

### 5.6 封装 Axios 请求

在 `src/utils/request.js` 中封装统一的 HTTP 请求：

```javascript
import axios from 'axios'

const request = axios.create({
  baseURL: '/api',
  timeout: 10000,
  headers: { 'Content-Type': 'application/json' }
})

// 请求拦截器 - 自动加上 token
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 响应拦截器 - 统一处理错误
request.interceptors.response.use(
  (response) => {
    const res = response.data
    if (res.code != 200) {
      if (res.code == 401) {
        localStorage.removeItem('token')
        window.location.href = '/login'
      }
      return Promise.reject(new Error(res.message || '请求失败'))
    }
    return res
  },
  (error) => {
    if (error.response && error.response.status === 401) {
      localStorage.removeItem('token')
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)

export default request
```

### 6.4 小程序全局逻辑 app.js

```javascript
const { BASE_URL, WS_URL } = require('./utils/constants')

App({
  globalData: {
    userInfo: null,
    token: null,
    baseUrl: BASE_URL,
    wsUrl: WS_URL,
    location: null
  },

  onLaunch() {
    this.checkLoginStatus()
  },

  checkLoginStatus() {
    const token = wx.getStorageSync('token')
    const userInfo = wx.getStorageSync('userInfo')
    if (token && userInfo) {
      this.globalData.token = token
      this.globalData.userInfo = userInfo
    }
  },

  setToken(token) {
    this.globalData.token = token
    wx.setStorageSync('token', token)
  }
})
```

## 九、核心业务流程串讲

### 9.1 用户停车流程

1. 用户打开小程序 → 获取定位 → 展示附近停车场
2. 选择停车场 → 查看车位分布 → 预约/直接前往
3. 车辆入场 → 后端创建 `parking_record` → 车位状态变"占用"
4. 车辆出场 → 计算费用 → 调用微信支付 → 订单完成

### 9.2 共享车位流程

1. 业主在小程序发布共享信息 → 创建 `share_record`
2. 其他用户租用 → 创建 `share_bill` 分账记录
3. 定时任务 `ShareRecordExpireTask` 自动检查到期记录

### 9.3 数据大屏实时更新

1. 后端通过 `ParkingWebSocketHandler` 推送实时数据
2. 管理后台的 Dashboard 连接 WebSocket
3. 车位状态变化时，大屏自动刷新图表和数字
