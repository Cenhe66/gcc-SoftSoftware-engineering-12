# 智慧停车场管理系统 - 功能流程与数据库结构

## 一、项目概述

本项目是一个基于微信小程序的智慧停车场管理系统，采用前后端分离架构：
- **前端**：微信小程序（`parking-miniapp`）
- **后端**：Spring Boot + MyBatis-Plus（`parking-server`）

核心功能涵盖：停车场浏览与导航、车位预约、共享车位、停车记录、挪车服务、支付结算、AR寻车、健康检查等。

#### 接口列表
| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/reservation/create` | POST | 创建预约 |
| `/api/reservation/cancel/{id}` | POST | 取消预约 |
| `/api/reservation/entry/{id}` | POST | 入场（扫码/手动） |
| `/api/reservation/exit/{id}` | POST | 离场，返回停车费金额 |
| `/api/reservation/pay/{id}` | POST | 支付预约费（状态 4→0） |
| `/api/reservation/pay-parking/{id}` | POST | 支付停车费（状态 6→2） |
| `/api/reservation/list/{userId}` | GET | 查询用户的预约列表 |
| `/api/reservation/detail/{id}` | GET | 预约详情 |

#### 预约状态（`reservation.status`）
| 值 | 含义 | 说明 |
|----|------|------|
| 0 | 待使用 | 已支付/免费，可入场 |
| 1 | 使用中 | 已入场停车 |
| 2 | 已完成 | 离场并结清费用 |
| 3 | 已取消 | 用户主动取消 |
| 4 | 待支付 | 刚创建订单，需支付预约费 |
| 5 | 已过期 | 超时未入场 |
| 6 | 待支付停车费 | 离场后有停车费待支付 |

#### 预约流程

```
【普通车位 / 共享车位 - 他人预约】
1. 选择车位 → 进入预约页
2. 选择时间 → 计算预约费（2元/小时）
3. 提交预约 → 创建订单（status=4 待支付）
4. 跳转支付页 → 支付预约费
5. 支付成功 → status=0（待使用），车位锁定（status=2）
6. 到达车场 → 扫码/点击入场（status=1，车位占用）
7. 停车结束 → 点击离场
8. 计算停车费 → 若停车费>0，status=6；若=0，status=2
9. 支付停车费 → status=2（已完成）
10. 车位释放（status=0）

【共享车位 - 他人预约（特殊）】
- 离场时停车费按业主设置的 shareRecord.hourlyPrice 计算
- 系统自动生成 share_bill（业主得80%，平台得20%）
- 业主余额实时增加

【私人车位 - 业主自用】
1. 选择自己的私人车位 → 进入预约页
2. 页面显示"个人车位，免费使用"
3. 提交预约 → 创建订单（status=0 直接待使用，reserveFee=0）
4. 入场（status=1）
5. 离场 → parkingFee=0，直接 status=2
6. 全程无支付流程

【取消/过期】
- 用户主动取消：status=3，释放车位
- 超时未支付（status=4）：自动取消，status=5
- 超时未入场（status=0）：自动过期，status=5
```

#### 费用计算规则
| 场景 | 预约费 | 停车费 | 说明 |
|------|--------|--------|------|
| 普通车位 | 2元/小时 | parkingLot.hourlyRate | 标准费率 |
| 共享车位 | 2元/小时 | shareRecord.hourlyPrice | 业主定价 |
| 私人车位（业主自用） | 0 | 0 | 完全免费 |
| 15分钟内离场 | 按实际 | 0 | 免费时长 |

### 2.4 共享车位功能（核心）

#### 涉及页面
- `share-space` - 我的车位（发布共享）
- `share-record` - 共享记录列表
- `share-bill` - 共享账单/收益

#### 接口列表
| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/share-record/publish` | POST | 发布共享 |
| `/api/share-record/pause/{id}` | PUT | 暂停共享 |
| `/api/share-record/resume/{id}` | PUT | 恢复共享 |
| `/api/share-record/stop/{id}` | PUT | 结束共享 |
| `/api/share-record/list` | GET | 查询共享记录列表 |
| `/api/share-record/stats` | GET | 共享统计 |
| `/api/share-bill/create` | POST | 创建共享账单（内部调用） |
| `/api/share-bill/list` | GET | 查询账单列表 |
| `/api/share-bill/stats` | GET | 账单统计 |

#### 共享记录状态（`share_record.status`）
| 值 | 含义 |
|----|------|
| 0 | 草稿/其他 |
| 1 | 共享中 |
| 2 | 已暂停 |
| 3 | 已结束 |
| 4 | 待停止（新增强化状态） |

#### 共享类型（`share_record.share_type`）
| 值 | 含义 |
|----|------|
| 0 | 按时计费 |
| 1 | 包月计费 |

#### 共享流程

```
【业主发布共享】
1. 进入"我的车位" → 选择空闲私人车位
2. 设置共享类型（按时/包月）、单价、共享时段
3. 提交 → share_record.status=1，parking_space.is_shared=1
4. 车位在地图上显示为紫色"共享"标签，标注价格

【他人预约共享车位】
- 同 2.3 预约流程，但停车费按业主定价计算

【业主暂停共享】
- 若当前无活跃预约（待使用/使用中/待支付）：直接暂停
- 若有活跃预约中：
  - 使用中 → 转为"待停止"状态（status=4），新预约被拒绝
  - 待使用/待支付 → 自动取消并全额退款，然后暂停
  - 待支付停车费 → 允许直接暂停，未结账单转平台催收

【业主恢复共享】
- 检查车位状态正常 → share_record.status=1

【业主结束共享】
- 与暂停逻辑类似，但状态变为已结束（status=3）
- 结束后 parking_space.is_shared=0

【自动过期】
- share_record.end_time 到期后，系统自动结束共享
- 若有活跃预约，强制结束（当前存在数据不一致风险）
```

#### 收益分成
| 项目 | 比例 |
|------|------|
| 业主收益 | 80% |
| 平台收益 | 20% |
| 到账方式 | 实时打入 user.balance |

## 六、补充说明

### 6.1 当前已知问题/注意点
1. **共享过期自动处理**：`autoExpire` 强制结束共享时未检查活跃预约，可能导致数据不一致
2. **共享车位挪车判断**：`ParkingSpaceController.info` 中的 `status == 3` 判断已失效（现在用 `is_shared == 1`）
3. **共享状态显示**：前端 `parking-detail` 已适配新的 `is_shared` 字段，显示紫色"共享"标签

### 6.2 权限说明
- 大部分查询接口无需特殊权限
- 用户操作类接口通过 JWT Token 鉴权，从 Header 中提取 `currentUserId`
- 管理员额外使用 `/api/admin/login` 接口
