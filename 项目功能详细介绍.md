# 智能停车管理系统 —— 各功能模块详细介绍

> 基于项目实际代码（SpringBoot后端 + 微信小程序 + Vue管理后台）梳理各功能实现方式

---

## 一、智能引导模块

### 1.1 动态车位热力图

**功能说明：**
实时展示停车场各车位使用状态，用不同颜色区分：红色（已占用）、绿色（空闲）、黄色（即将离场）。

**实现思路：**
- 后端通过 WebSocket 定时扫描车位状态表，把变化推送给前端
- 前端管理后台用 ECharts 热力图渲染，管理大屏实时刷新
- 小程序端在停车场详情页用简化版颜色块展示

### 1.2 AR实景寻车

**功能说明：**
车主在大型停车场找不到自己车时，开启AR寻车，相机画面上叠加箭头指引方向。

**实现思路：**
- 车辆入场时记录车位坐标（x, y, floor）
- 用户输入车位号后，系统根据预存坐标计算相对方向
- 小程序 `<camera>` 组件获取实时画面
- 陀螺仪（`wx.startGyroscope`）获取手机朝向
- Canvas 叠加层根据方向计算绘制箭头

### 1.3 离场预判

**功能说明：**
预判哪些车位未来1-5分钟内会空出，标记为黄色提醒其他车主。

**实现思路：**
- 后端定时任务（Spring Task）每分钟扫描当前停车记录
- 规则引擎综合判断：停留时长、预约结束时间、历史离场规律
- 命中规则的记录标记为"即将离场"
- 通过 WebSocket 推送给前端

### 1.4 个性化推荐

**功能说明：**
根据用户历史行为画像（常用楼层、停车习惯），入场时推荐最优车位。

**实现思路：**
- user表记录用户画像标签
- 进场时根据 user_id 调取画像，匹配空闲车位
- 加权排序后推荐前3个最优车位

## 二、预约管理模块

### 2.1 长周期分时段预约

**功能说明：**
用户可提前1-7天预约车位，自定义起止时间（精确到半小时）。

**实现思路：**
- reservation 表存预约记录（user_id、space_id、start_time、end_time）
- 创建预约时检查时段冲突（同一车位时段不能重叠）
- 预约成功状态置为"待使用"，对应车位预留

### 2.2 预约超时智能管控

**功能说明：**
预约开始后15分钟未入场自动释放车位，超时未到且未取消执行惩罚扣费。

**实现思路：**
- `ReservationReleaseTask.java` 定时任务每分钟扫描
- 当前时间 > 预约开始时间 + 15分钟 且 未入场 → 自动释放
- 超时未取消 → 扣费记录，状态置为"已违约"
- 结合 Redis 过期事件双保险

## 三、车位共享模块

### 3.1 私有车位分时共享

**功能说明：**
业主可发布自己的车位，按时段对外出租，支持按时/包月两种模式。

**实现思路：**
- share_record 表存共享记录（owner_id、space_id、time_slot、price、status）
- 业主在小程序"我的共享"页面发布、暂停、结束
- 其他用户可在空闲时段看到并使用

### 3.2 智能地锁联动

**功能说明：**
预约/共享成功后地锁自动降下，车辆离场地锁自动升起。

**实现思路：**
- parking_space 表加 lock_status 字段
- 状态变更时通过 WebSocket 下发指令（演示用软件模拟）
- 实际生产可对接IoT地锁设备

### 3.3 共享分账管理

**功能说明：**
系统自动统计共享使用时长、计费金额，生成业主与平台分账账单。

**实现思路：**
- share_bill 表存分账记录（share_id、total_amount、owner_share、platform_share）
- 共享订单完成时自动计算分账
- 业主可按日/月导出账单

## 四、车辆健康检测模块

### 4.1 多项异常检测

**功能说明：**
检测车窗未关、车灯常亮、轮胎气压异常等隐患。

**实现思路：**
- health_check_log 表存检测记录
- 后端定时任务模拟传感器数据（毕设环境无硬件）
- 规则引擎：胎压 < 2.0bar 报警、车灯开启时长 > 30分钟报警等
- 检测到异常通过小程序订阅消息推送预警

## 五、匿名挪车模块

### 5.1 扫码挪车发起

**功能说明：**
被挡车主扫描对方车辆二维码或输入车牌发起挪车请求，双方手机号都隐藏。

**实现思路：**
- move_car_request 表存请求记录（屏蔽真实手机号，只存 openid）
- 用户脱敏展示：车牌"京A***12"形式
- WebSocket 建立实时通讯通道

### 5.2 实时消息对话

**功能说明：**
双方通过预设模板（"请尽快挪车""马上到"等）实时沟通。

**实现思路：**
- move_car_message 表存消息记录
- WebSocket 长连接，新消息实时推送
- 快捷回复模板减少输入成本

## 六、缴费与访客模块

### 6.1 无感缴费

**功能说明：**
车辆离场自动识别车牌、核算费用、发起扣款，抬杆放行。

**实现思路：**
- parking_record 记录入场/离场时间
- 费用计算规则：基础价 + 时长 × 单价
- payment_order 表存订单（创建订单 → 模拟支付回调 → 状态变更为已支付）
- 演示环境走沙箱模拟，流程与真实支付一致

### 6.2 临时访客授权

**功能说明：**
业主可生成限时通行码或临时车牌白名单，分享给访客。

**实现思路：**
- visitor_auth 表存授权记录（visitor_name、plate_number、valid_from、valid_to、max_uses）
- 生成唯一授权码，小程序可分享
- 到期自动失效，超次数自动失效

## 七、管理后台模块

### 7.1 数据大屏（Dashboard）

**功能说明：**
运营方一屏看全场：今日停车次数、收益、空闲车位、在线用户、24小时车流、7天收益趋势、热力图、TOP5排行、告警信息。

**实现思路：**
- `Dashboard.vue` 整合多个 ECharts 组件
- 后端 `/api/dashboard/stats` 等接口提供聚合数据
- 30秒自动刷新一次
- 1920x1080 响应式布局

**核心组件：**
- `StatCard.vue`：统计卡片（4个）
- `TrafficChart.vue`：24小时车流折线图
- `RevenueChart.vue`：7天/30天收益柱状图
- `HeatmapChart.vue`：楼层×时间热力图
- `ParkingRank.vue`：停车场收益/使用率 TOP5
- `AlertList.vue`：系统告警滚动列表

### 7.2 停车场管理

**功能说明：**
CRUD 停车场信息，查看车位使用率。

**实现思路：**
- `ParkingManage.vue` 表格 + 表单
- Element Plus 组件，分页 + 搜索
- 车位使用率 = 已用车位 / 总车位

### 7.3 停车记录管理

**功能说明：**
查看所有停车记录，按日期/状态筛选，支持导出和详情查看。

**实现思路：**
- `RecordManage.vue` 表格 + 日期范围选择器
- 后端支持分页查询、多条件过滤
- 详情弹窗显示完整记录

### 7.4 用户管理

**功能说明：**
管理用户信息、角色、状态（启用/禁用）。

**实现思路：**
- `UserManage.vue` 表格 + 表单
- Element Plus Switch 组件控制状态
- 角色字段控制权限（admin / user）

### 7.5 个人中心

**功能说明：**
管理员查看/编辑个人信息，修改密码。

**实现思路：**
- `Profile.vue` 表单 + 头像上传
- 修改密码需校验旧密码

### 7.6 系统设置

**功能说明：**
基本设置、停车设置、通知设置、数据备份。

**实现思路：**
- `Setting.vue` 多 Tab 页面
- 配置项存数据库或配置文件

## 八、核心技术栈

| 端 | 技术 |
|----|------|
| 后端 | SpringBoot 2.x + MyBatis-Plus + MySQL 8.0 + Redis 6.x |
| 实时通信 | Spring WebSocket |
| 任务调度 | Spring Task |
| 用户端 | 微信原生小程序（WXML + WXSS + JS） |
| 管理端 | Vue 3 + Element Plus + ECharts 5 + Pinia + Vite |

## 九、整体数据流

```
用户操作（小程序）
       ↓
HTTP/WebSocket 请求
       ↓
SpringBoot Controller 接收
       ↓
Service 业务处理
       ↓
Mapper 操作 MySQL / Redis
       ↓
返回结果 / 推送 WebSocket 消息
       ↓
小程序或管理后台渲染
```
