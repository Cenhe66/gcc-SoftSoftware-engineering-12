## 1. 系统概述

本系统是一个面向城市停车的智慧解决方案，旨在解决"停车难、找车难、挪车难"三大痛点。系统为车主提供从**找停车场 → 预约车位 → 入场停车 → 离场缴费 → 寻车**的全流程服务，同时支持**车位共享**和**挪车通知**等社交化功能。

**核心解决的用户痛点：**

| 痛点          | 解决方案                    |
| ----------- | ----------------------- |
| 不知道附近哪里有空车位 | 实时停车场列表 + 空闲车位数量 + 距离排序 |
| 到了停车场没车位    | 在线预约车位，锁定车位             |
| 忘了车停在哪      | AR 寻车导航 + 楼层切换          |
| 车被堵住出不来     | 扫码挪车 + 实时消息沟通           |
| 私家车位闲置浪费    | 车位共享 + 收益计算             |
| 访客停车不便      | 访客授权码一键生成               |
| 车辆健康担忧      | 车辆健康检测                  |

## 2. 技术架构

### 前端（微信小程序）

- **框架：** 微信原生小程序框架
- **语言：** JavaScript + WXML + WXSS
- **网络请求：** 封装 wx.request，支持请求/响应拦截器、自动重试机制、Token 管理
- **实时通讯：** WebSocket（移车消息推送）
- **设备能力：** 相机（AR寻车/扫码）、陀螺仪（AR方向）、定位（附近停车场）、扫码（挪车）
- **UI 风格：** 蓝白主色调，Vant 风格组件

### 后端（Java）

- **框架：** SpringBoot
- **API 风格：** RESTful，统一 `Result<T>` 响应格式
- **数据：** MySQL（Spring Data JPA / MyBatis）
- **功能模块：** 用户管理、停车场管理、车位管理、预约管理、支付订单、挪车请求、共享记录、健康检测、离车预测

### 接口设计

```
前端 (小程序)  →  HTTP/WebSocket  →  后端 (SpringBoot :8080)  →  MySQL
```

### 3.1 用户登录与认证

**页面：** `pages/login/login`

提供两种登录方式：

| 方式              | 说明                                |
| --------------- | --------------------------------- |
| **手机号 + 验证码登录** | 输入手机号，获取短信验证码，6位验证码校验             |
| **微信授权登录**      | 调用 wx.login 获取临时 code，后端换取 openId |

**关键设计：**

- 登录前必须勾选《用户协议》和《隐私政策》
- 登录成功后 Token 和用户信息存入 Storage，全局可用
- 已登录状态下自动跳转首页
- 发送验证码有 60 秒倒计时限制

**后端接口：** `POST /api/user/login`

### 3.2 首页停车场列表

**页面：** `pages/index/index`

**功能描述：**

- 获取用户当前位置（微信定位 API）
- 加载附近停车场列表
- 按**距离从近到远**排序
- 显示停车场名称、地址、空闲车位数、总车位数、每小时价格、距离
- 下拉刷新、上拉加载更多
- 支持搜索停车场

**三种入口模式：**

1. 正常浏览 → 点击进入停车场详情
2. 选择模式（从预约/共享页面跳转过来选择停车场）
3. 搜索模式（模糊搜索停车场名称）

**后端接口：** `GET /api/parking-lot/list`

### 3.3 停车场详情与车位图

**页面：** `pages/parking-detail/parking-detail`

**功能描述：**

- 展示停车场详细信息（名称、地址、总车位、空闲车位、价格、免费时长等）
- **多层车位可视化**：按楼层（B1/B2/B3）展示车位分布图
- 车位状态用颜色区分：
  - 🟢 绿色 = 空闲（可预约）
  - 🔴 红色 = 已占用
  - 🟠 橙色 = 已预约
  - 🔵 蓝色 = 共享中
- 点击空闲车位 → 弹窗确认 → 跳转预约页面
- "导航到停车场"调用微信内置地图导航

**后端接口：**

- `GET /api/parking-lot/detail/{id}`
- `GET /api/parking-space/list?lotId=`

### 3.4 车位预约

**页面：** `pages/reservation/reservation`

**功能描述：**

- 选择预约日期（可预约未来 7 天）
- 选择开始时间和结束时间
- 自动计算**预约时长**和**预约费用**（2元/小时）
- 输入车牌号（自动大写）和手机号
- 表单校验通过才允许提交
- 提交后跳转支付页面

**预约状态流转：**

```
待支付 → 待使用 → 使用中 → 已完成
                  ↘ 已取消
                  ↘ 已超时
```

**后端接口：** `POST /api/reservation/create`

***

### 3.5 预约列表与管理

**页面：** `pages/reservation-list/reservation-list`

**三个 Tab 分类：**

| Tab     | 说明          | 操作             |
| ------- | ----------- | -------------- |
| **待使用** | 已支付但未入场的预约  | 取消预约、导航去停车场    |
| **使用中** | 已入场正在停车     | 查看已停时长/预估费用、离场 |
| **历史**  | 已完成/已取消/已超时 | 查看详情           |

**核心逻辑：**

- **怠时实时更新**：使用中状态下，每 30 秒自动刷新已停时长和预估费用
- **支付倒计时**：未支付订单有 60 秒支付窗口，超过自动转为"已超时"
- **入场离场流程**：
  - 入场：确认 → 调用入场接口 → 开始计时
  - 离场：确认 → 调用离场接口 → 计算停车费 → 跳转支付
- **15 分钟免费**：停车不满 15 分钟免停车费

**后端接口：**

- `GET /api/reservation/list/{userId}`
- `POST /api/reservation/cancel/{id}`
- `POST /api/reservation/entry/{id}`
- `POST /api/reservation/exit/{id}`

### 3.7 支付系统

**页面：** `pages/payment/payment`

**支持的支付类型：**

| 类型        | 说明            |
| --------- | ------------- |
| **预约费支付** | 预约车位时预先支付的预约费 |
| **停车费支付** | 离场时结算的停车费     |

**支付方式：**

1. **微信支付** — 调用微信支付接口（wx.requestPayment）
2. **余额支付** — 账户余额直接扣款（调用后端扣款接口）

**流程：**

```
确认支付 → 构建订单 → 调后端创建支付单
  ├─ 微信支付 → 调起微信支付 → 支付回调 → 成功
  └─ 余额支付 → 后端处理 → 成功
```

**后端接口：**

- `POST /api/payment/create`
- `POST /api/payment/callback`

### 3.8 车位共享

#### 发布共享车位

**页面：** `pages/share-space/share-space`

**功能描述：**

- 选择停车场（从列表选择）
- 输入车位编号、选择车位类型（普通/充电）
- **两种时间模式：**
  - **固定时段**：选择起止日期，在该时间段内共享
  - **每日重复**：选择每周哪些天共享
- 设置每小时价格，系统自动给出建议价
- **收益计算器**：实时显示每日预估收益和总收益（扣除 20% 平台服务费）
- 平台抽成 20%，车主实际收入 80%

**后端接口：** `POST /api/share-record/publish`

#### 共享记录管理

**页面：** `pages/share-record/share-record`

**功能描述：**

- 显示共享统计数据（总共享次数、当前活跃数、总收益）
- 两个 Tab：进行中 / 已结束
- **状态管理操作：**
  - **暂停共享** — 暂时停止接受预约
  - **恢复共享** — 重新开放
  - **结束共享** — 彻底结束（不可恢复）
- 查看共享车位二维码（打印贴在车位上供扫码）

**后端接口：**

- `GET /api/share-record/list`
- `PUT /api/share-record/pause/{id}`
- `PUT /api/share-record/resume/{id}`
- `PUT /api/share-record/stop/{id}`

### 3.10 挪车服务

挪车服务是本系统的一大亮点功能，解决"车被堵住"的常见痛点。

#### 整体流程

```
被堵车主 → 扫码/输入车位号 → 选择原因 → 发送请求
  → 车主收到订阅消息 → 双方实时聊天沟通 → 完成挪车
```

#### ① 挪车主页

**页面：** `pages/move-car/move-car`

**三种触发方式：**

1. **扫码识别** — 扫描车位二维码自动识别车位
2. **手动输入** — 选择停车场 + 输入车位号搜索
3. **最近记录** — 显示最近 3 条挪车记录快速入口

#### ② 挪车请求

**页面：** `pages/move-car-request/move-car-request`

**功能描述：**

- 显示对方车辆信息（车牌号脱敏处理：京A\*\*345）
- 选择挪车原因（5 个预设原因）
- 可填写补充说明和联系方式
- 提交前请求**订阅消息授权**，以便通知车主
- 提交后跳转到消息页面

#### ③ 挪车消息实时聊天

**页面：** `pages/move-car-message/move-car-message`

**功能描述：**

- 基于 **WebSocket** 的实时消息通信
- 发送文字消息
- 快捷回复（"马上来挪车"、"请稍等5分钟"等）
- 消息气泡样式，区分自己/对方
- 显示请求状态（进行中/已结束）

**后端接口：**

- `GET /api/move-car/request/{id}`
- `GET /api/move-car-msg/list/request/{id}`
- `POST /api/move-car-msg/send`
- WebSocket 订阅 `move_car_{requestId}` 主题

#### ④ 挪车记录

**页面：** `pages/move-car-list/move-car-list`

**功能描述：**

- 合并显示服务端记录 + 本地匿名记录
- 按时间倒序排列
- 支持未登录状态使用（本地存储）
- 点击进入对话详情

**后端接口：** `GET /api/move-car/list/requester/{userId}`

### 3.15 服务器端功能

#### 仪表盘（Dashboard）

**接口：** `GET /api/dashboard/data`

提供管理后台的统计数据总览，包括停车总数、收入总额、车位使用率等运营指标。

#### 热力图（Heatmap）

**接口：** `GET /api/heatmap/lot/{lotId}`

按停车场展示车位使用热力数据，反映各区域繁忙程度，帮助管理人员优化车位分配。

#### 离车预测（Departure Prediction）

**接口：**

- `GET /api/prediction/record/{recordId}` — 预测单个车辆的预计离开时间
- `GET /api/prediction/lot/{lotId}` — 预测停车场的整体空位释放趋势

基于历史停车数据（平均停车时长、时段规律等）建模，预测车位何时会被释放，为即将到达的车主提供参考。

#### 健康检测日志（Health Check Log）

记录每次车辆检测的结果，支持查询历史检测记录。

## 5. 业务流程总结

### 🔄 完整停车流程

```
打开小程序 → 浏览附近停车场 → 查看详情和车位图
  → 预约车位 → 支付预约费 → 到场 → 确认入场
  → 开始停车（实时计时） → 离场 → 支付停车费 → 完成
```

### 🔄 车位共享流程

```
发布共享（选择停车场 + 时段 + 价格）→ 系统展示
  → 其他用户预约共享车位 → 收益入账 → 账单查看
```

### 🔄 挪车流程

```
扫码/输入车位号 → 选择原因 → 发送挪车请求
  → 实时聊天沟通 → 挪车完成 → 结束
```

### 🔄 寻车流程

```
进入停车场 → 打开AR寻车 → 相机画面 + 方向指引
  → 切换楼层 → 跟随导航 → 找到车辆
```
